# 溯源 · 新意坐标仪 — 设计文档

> 日期：2026-06-24（进度更新 2026-06-25）· 状态：已上线 idea-track.vercel.app，详见文末「实现进度」

## 一句话

扔进一个观点（别人的爆款 / 你自己随口说的），它把这观点摆到人类思想地图上，告诉你**新不新、源头在哪、旁边哪还空着**。不造爆款、不写标题——只给「背后那个东西」。

## 命题（为什么做）

未来没人看完整内容，大家只问 AI「这讲了个啥」，最后只剩**观点**——于是变成观点对碰。**让一个观点脱颖而出的，唯一是它「真的新」（长出新枝），不是包装。** 我们天天靠抄爆款吃饭，但真正稀缺、值得追的是：这观点的源头是什么？它到底有没有新意？我自己的观点有没有新意？

核心信念（来自团队）：**AI 本身就是人类思想的图谱，我们只要找到把它显性映射出来的方式 + 给它上护栏。** 所以不预建知识图谱数据库，而是按观点临时把模型权重里的隐性思想图谱「显性化」，再用防幻觉纪律保证可信。

## 非目标（明确不做）

- 不做赛道分析 / 全网空白雷达（老板想要的商业层，留作 v2）。
- 不做全网内容采集管线、不接灰产数据 API。
- 不生成爆款标题 / 内容 / 呈现包装。

## 核心链路（单观点，五段）

1. **逼问（elicit）**【抬杠的同谋】最多 2 轮苏格拉底追问，把模糊感受逼成「有对立面 + 有边界」的可证伪主张。可跳过。
2. **结构化（structure）**【谦卑的镜子】抽成四字段：主张 / 断言 / 论证 / 适用范围。
3. **溯源（trace）**纵向多层血缘假设链 + 横向多阵营众源，每条带置信度与假设语气词。
4. **判定（verdict）**五选一关系分类（复述 / 应用 / 精化 / 综合 / 新分支），仅 4、5 及很强的 3 触发 🌱。
5. **前沿罗盘（compass）— 唯一新增的核心段**
   - **罗盘为主**：诚实指出旁边哪还空着（反向张力 / 未焊接的一对 / 还空着的下一步 / 被忽略的边界 / 过时的前提），把思考留给用户。
   - **军师可选（expand）**：用户主动点某个缺口 → AI 才把它展开成具体新观点候选，明确标注「生成草稿、非定论」。

前四段提示词原样移植自原型（`reference/原型溯源/溯源.html` 的 PROMPTS），语义未改写；后两段沿用同样人格与防幻觉纪律新写。见 `src/lib/prompts.ts`。

## 防幻觉三重设防（继承）

1. 假设语气词白名单（可能来自 / 线索指向 / 一种血缘假设 / 推测经由），禁断言式（出自 / 引自）。
2. 置信度圆点 0–5（●●●○○），纯猜测转为 `insufficient` 留白节点，宁可留白不编造人名/书名/年份。
3. 判关系不判人；前沿「拥挤/空旷」判断须注明「仅基于思想史认识，非实时市场数据」。

## 架构与技术栈

- Next.js 16（App Router, TS, Tailwind v4）+ MiniMax-M3（OpenAI 兼容端点 `api.minimaxi.com/v1`）。
- 引擎：六个 thin API 路由（`/api/{elicit,structure,trace,verdict,compass,expand}`），前端编排，逐段显示进度。
- `src/lib/minimax.ts`：M3 是推理模型，正文内嵌 `<think>…</think>`，解析前剥离 + JSON 兜底重试。
- `runtime=nodejs`、`maxDuration=300`（M3 推理较慢）。
- 密钥走环境变量（`MINIMAX_API_KEY` / `MINIMAX_BASE_URL` / `MINIMAX_MODEL`），`.env*` 不提交。

## 第一纵切范围（本次实现）

`单观点输入 →（可跳过的逼问）→ 结构化 → 溯源 → 判定 → 前沿罗盘（+ 军师展开）→ 网页报告`。两个模式（拆别人的 / 检验我自己的）先合一，仅影响文案。

## 已验证

`structure` 与 `trace` 对 MiniMax-M3 实跑通过：JSON 解析稳、血缘带假设语气词与置信度、防幻觉纪律守住（详见会话验证）。原型阶段另有 4 案例对抗式核查报告（3 复述 / 1 综合🌱，零编造），见 `reference/原型溯源/验证报告.md`。

## 实现进度（截至 2026-06-25）

已上线 **idea-track.vercel.app**（Vercel 项目 `idea-track`，GitHub `Mike1075/idea_track`）。在 MVP 五段链基础上又做了：

- **判定 = 3 次采样投票 + 三档稳定度**（复述区 / 边界区 / 新枝区）。起因：实测发现单次判定不稳——同一观点跨模型+同模型多次跑会判出 2/3/4/5（判定是最不稳的输出；血缘锚点与前沿缺口类型才是跨模型最稳的）。`src/lib/verdict.ts`，verdict 升温 0.7 跑 3 次，分歧本身作信号。
- **导出**：报告一键拷贝 Markdown / 下载 .md（`src/lib/export.ts`）。
- **访问口令闸门**（`src/proxy.ts`，`APP_ACCESS_CODE`，前端 `x-access-code` 头）：保护 /api 不被陌生人刷爆 MiniMax/网关额度。
- **图片识别输入**：M3 本身多模态，直接 OCR 截图 → 观点文本（`/api/ingest/image`，无需额外 vision 模型）。
- **URL 抓取**：公众号正文 / 通用网页可用；YouTube 字幕已被平台锁死（PoToken/attestation），仅尽力而为（`src/lib/fetchers.ts`）。
- 环境变量：`MINIMAX_API_KEY` / `MINIMAX_BASE_URL`(api.minimaxi.com/v1) / `MINIMAX_MODEL`(MiniMax-M3) / `AI_GATEWAY_KEY`(Vercel AI Gateway，跨模型/待用) / `APP_ACCESS_CODE`。

## 后续（按优先级）
